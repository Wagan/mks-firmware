# tools/ — хост-инструменты для работы с МКС

Скрипты на Python, запускаются на ПК (не на микроконтроллере). Общаются с платой
МКС по USB CDC (Virtual COM Port). Источник истины по протоколу —
`docs/PROTOCOL_SPEC.md`.

## Состав

- `mks_protocol.py` — библиотека протокола: CRC8, сборка/разбор кадров, класс
  `MKS` (обёртка над COM-портом), парсеры ответов. Переиспользуется тестами.
- `mks_ping_test.py` — проверка связи: PING / INIT / GET_STATUS.
- `requirements.txt` — зависимости (pyserial).

## Установка зависимостей

```
pip install -r requirements.txt
```

## Запуск проверки связи

```
python mks_ping_test.py COM3
```

Замени `COM3` на порт, под которым система видит «STM32 Virtual COM Port»
(Диспетчер устройств → Порты (COM и LPT)).

Ожидаемо:
- PING → STATUS=OK, DATA = 0x00 (PONG)
- INIT → STATUS=OK (если инициализированы все модули DWM1000)
- GET_STATUS → STATUS=OK, 7 байт полей состояния

## Заметки

- Плата должна быть прошита прошивкой со слоем протокола (Шаг 2+).
- Если PING не отвечает: проверь, что открыт правильный COM-порт, плата запитана,
  прошивка актуальна.
- Новые тесты по мере роста протокола импортируют `mks_protocol.py`, не дублируя
  CRC/кадрирование.
