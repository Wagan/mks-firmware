## Что за проект
Прошивка для платы **МКС** (Модуль коммуникации и сопряжения) — плата расширения
ПК на **STM32F411RETx** с **двумя модулями DWM1000** (UWB, DecaWave DW1000).
МКС принимает команды от MATLAB по **USB CDC (Virtual COM)**, транслирует их в
операции с регистрами DW1000, собирает диагностику (RSSI, SNR, CIR, метки времени).
Задача НИР — обнаружение СШП/UWB сигналов.

Есть вторая плата — **мини-стенд** (Nucleo-F411RE, 1× DWM1000) — сейчас **ОТЛОЖЕНА**,
её код держим в заготовках (`#ifdef BOARD_MINISTEND`), но не разрабатываем.

## Текущий этап
**Bring-up, BARE-METAL (без FreeRTOS).** Цель — поднять SPI и прочитать `DEV_ID`
с каждого DW1000 (ожидаем `0xDECA0130`). FreeRTOS добавим через CubeMX ПОЗЖЕ,
когда дойдём до протокола и многозадачности. Сейчас не вводить RTOS.

## КРИТИЧЕСКИ ВАЖНЫЕ факты по железу МКС (из схемы PCI_E_PCB)
- **У каждого DWM1000 СВОЯ шина SPI:** M1 → **SPI2**, M2 → **SPI3**.
  SPI1 занят внешней EEPROM M95080 (это НЕ радио).
  => НЕ общая шина (вопреки текстовым документам — верить схеме).
- **Перекрёстная нумерация IRQ:** IRQ1 → M2, IRQ2 → M1. Именовать по модулю.
- **RST через N-MOSFET (BSS138)** — полярность, вероятно, инвертирована.
  Вынесена в `DW_RST_ACTIVE_HIGH` в board_config.h. Проверяется на железе (OQ-9).
- HSE кварц **24 МГц**. USB FS требует 48 МГц (настройка PLL в CubeMX).
- USB CDC на PA11/PA12.
- На МКС **нет** физической DEBUG-консоли — всё через MATLAB. Отладка через SWD.

## DecaDriver — как работать с двумя чипами
- Драйвер хранит состояние в массиве `dw1000local[DWT_NUM_DW_DEV]`.
- **Передать компилятору `-D DWT_NUM_DW_DEV=2`** (настройка CubeIDE), НЕ править
  vendor-файлы.
- Перед операциями с чипом: `deca_port_select_device(idx)` — он делает
  `dwt_setlocaldataptr(idx)` И запоминает шину/CS. Затем обычные `dwt_*` вызовы.
- Платформенные функции, которые мы реализуем (в deca_port.c):
  `writetospi`, `readfromspi`, `deca_sleep`, `decamutexon/off`.
- **ИСКЛЮЧЕНИЕ из правила «не трогать vendor»:** в `deca_device.c` исправлен баг
  vendor-драйвера в `dwt_setlocaldataptr()` — была инвертированная проверка границ
  (`if (DWT_NUM_DW_DEV > index)`), из-за чего функция возвращала `DWT_ERROR` для
  любого валидного индекса и ломала выбор устройства. Исправлено на
  `if (index >= DWT_NUM_DW_DEV)`, помечено маркером `NCPR FIX`. Это единственная
  санкционированная правка в `Drivers/decadriver/`; **при обновлении DecaDriver
  правку нужно перенести заново.**

## Решения по протоколу (для будущих этапов)
- CRC8: полином 0x07, init 0x00, без рефлексии/xorout. Покрытие — **LEN+CMD_ID+PARAMS
  (БЕЗ SYNC)**. SYNC=0xAA55. Ответ: CRC по LEN+STATUS+DATA.
- При ошибке CRC → вернуть код ошибки (INTERNAL_ERROR), не игнорировать молча.
- Параметры на проводе — little-endian. preamble_length/data_rate/PRF/PAC приходят
  как сырые числа → транслировать в enum DecaDriver (DWT_PLEN_*, DWT_BR_*, и т.д.)
  с валидацией (иначе INVALID_PARAM). 17 команд: CMD_ID 0x00–0x61.

## Правила работы
- НЕ модифицировать файлы в `Drivers/decadriver/`.
- Пины/периферию меняет пользователь в CubeMX (.ioc). Claude Code код CubeMX не
  перегенерирует — пишет логику в App/ и порт.
- Наш код в App/ — вне зоны генерации CubeMX (не затирается).
- Стиль: C, HAL. Комментарии можно по-русски.
