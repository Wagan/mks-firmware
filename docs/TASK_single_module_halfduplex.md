# Однокабельная сборка MKS_SIMPLEX: полудуплекс на одном модуле — заход 1 (§15.1)

**Дата:** 2026-07-22. **Основание:** RECON_single_module_halfduplex.md (`5e9399b`),
PLAN согласован (`10fce34`, ORDER: A `#define MKS_SIMPLEX`; B forcetrxoff M1 + не
конфигурить M1). **Гипотеза:** M1 (TX) глушит M2 (RX) на одной плате. **Заход 1 —
минимальная проверка гипотезы, БЕЗ Master/Slave-дисциплины** (коллизии — заход 2).
Меняется только радио-уровень прошивки; прикладной слой станции/GUI не тронут.

## Объём захода 1 (по уточнению архитектора)
Только: (1) сборка `MKS_SIMPLEX` (M2 оба направления, M1 заглушён) + (2) связка
полудуплекса на одном чипе (обязательный `rxenable` после TX — главный риск). Master/
Slave/опрос — заход 2 после подтверждения гипотезы. Двусторонний обмен для замера
дальности НЕ нужен — проверяется асимметрично (одна шлёт / одна слушает, как с китами).

## Шаг 1 — `board_config.h` (коммит)
Секция `BOARD_MKS`: раскомментируемый `#define MKS_SIMPLEX` →
```c
#ifdef MKS_SIMPLEX
  #define DW_RX_LISTEN_DEV  DW_DEV_M2
  #define DW_TX_SOURCE_DEV  DW_DEV_M2     /* оба направления на ОДИН чип */
  #define DW_SINGLE_MODULE  1
#else  /* двухмодульная как сейчас */
  #define DW_RX_LISTEN_DEV  DW_DEV_M2
  #define DW_TX_SOURCE_DEV  DW_DEV_M1
  #define DW_SINGLE_MODULE  0
#endif
```
`DW_DEVICE_COUNT` = 2 (M2 = индекс 1; INIT должен доходить). Закомментирован по
умолчанию → двухмодульная сборка байт-в-байт как раньше. `BOARD_MINISTEND` тоже помечен
`DW_SINGLE_MODULE=1` (один чип).

## Шаг 2 — прошивка `protocol.c` (коммит, всё под `#if DW_SINGLE_MODULE`)
- **Макросы `HALFDUPLEX_RX_OFF()`/`HALFDUPLEX_RX_ON()`** = `dwt_forcetrxoff` / `dwt_rxenable`
  при `DW_SINGLE_MODULE=1`, пустые при 0.
- **Связка полудуплекса (роли §1 сводятся на M2 бесплатно через `#define`):**
  - `HandleTX_FRAME`: `RX_OFF` перед `starttx`, `RX_ON` после TXFRS **и на ошибках/
    таймауте** (главный риск §7 — RX не остаётся выключенным).
  - `PROTOCOL_PollTx` (периодика/beacon): та же обвязка.
  - `HandleTX_STOP`: `RX_ON` после `forcetrxoff` (чип продолжает слушать).
- **Заглушение M1** (`HandleINIT`, после обнаружения живых): `forcetrxoff` всех живых
  модулей `!= DW_RX_LISTEN_DEV`. На плате только с M2 — цикл пуст. `entersleep` — запасом
  (в комментарии), если наводка останется.
- **Не конфигурить M1** (`HandleSET_PHY_CONFIG`): конфигурим ТОЛЬКО `DW_RX_LISTEN_DEV`
  (иначе `dwt_configure` поднимет PLL M1 → снова наводка).
- Синтакс-чек ОБЕИХ сборок (`BOARD_MKS` и `-DMKS_SIMPLEX`) `-Wall -Wextra` — чисто.

## Шаг 3 — доки (этот коммит)
HANDOFF §15.1 «Правка 4 (заход 1)»; этот отчёт. (Провод протокола не менялся — SPEC не
трогаем; изменение внутреннее, радио-уровень.)

## Почему это должно помочь (гипотеза)
- **До:** станция на одной плате держит M2 в непрерывном RX И активно передаёт M1 →
  M1 (5 см от M2, сильный сигнал) глушит собственный приём M2 → −94 dBm, обрывы.
- **После:** работает ОДИН модуль M2, TX и RX на нём разнесены во времени
  (forcetrxoff→starttx→rxenable), M1 полностью выключен (forcetrxoff, не сконфигурен) →
  нет самоглушения. Условия приближены к EVK (один чип, полудуплекс).

## Риски (учтены)
- **RX остался выключен после TX** — закрыт: `rxenable` во всех TX-точках, включая
  ошибочные ветки.
- **Гонка PollTx/PollRadio** — по факту нет: main loop однопоточный последовательный
  (`main.c:120–122`); TX-операция атомарна (busy-wait блокирует loop).
- **Потеря кадра во время передачи** — неотъемлемо для полудуплекса, приемлемо.
- **Наводка M1 через PLL** — митигирована (M1 не конфигурится + forcetrxoff); запас
  `entersleep`.
- **Двухмодульную не сломать** — весь новый код под `#if DW_SINGLE_MODULE`; проверено
  синтакс-чеком обеих сборок.

## Приёмка на железе (ВАГАН) — асимметричный замер дальности
1. **Собрать `MKS_SIMPLEX`**: раскомментировать `#define MKS_SIMPLEX` в `board_config.h`,
   пересобрать, прошить ОБЕ платы (или одну + слушатель).
2. **Асимметрично** (без встречных beacon'ов, чтобы не перебивали друг друга):
   - плата A (`MKS_SIMPLEX`) — **передатчик**: `txperiodic <payload>` (или beacon станции).
   - плата B (`MKS_SIMPLEX`) — **только слушатель**: приём, без своих передач.
3. **Замер устойчивой дальности** A→B: ожидание — **скачок к EVK-подобной 5–8.5 м**
   против прежних ~2 м (если гипотеза самоглушения верна). RSSI у B на фикс. дистанции.
4. Проверить: на близи (1–2 м) обрывы исчезли? На 5 м — стабильный приём с рабочим CIR
   (как EVK)?
5. Регресс двухмодульной: закомментировать `MKS_SIMPLEX`, пересобрать — loopback/CIR/
   метрики/GUI работают как раньше.

**Отчёт Вагана:** устойчивая дальность одномодульной A→B vs 2 м vs EVK 8.5 м; исчезли ли
обрывы на близи; нужен ли `entersleep` (осталась ли наводка). Если дальность прыгнула —
гипотеза подтверждена, идём в заход 2 (Master/Slave, устранение коллизий, двусторонний
обмен). Если нет — гипотеза самоглушения неверна, ищем дальше.
