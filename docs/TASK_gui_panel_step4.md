## 2. Кнопка «Включить»/«Выключить» (подключение COM)

- На инфо-панели. Заменяет кнопку «Подключить» с вкладки Настройки.
- Поле ввода COM-порта ОСТАЁТСЯ на вкладке Настройки (редко меняется — не засорять
  панель). Кнопка на панели использует значение порта из этого поля.
- Логика: «Включить» → открыть порт (+ страховка при подключении: flush → SET_STREAM_MODE 0
  → flush, как в Шаге 1). После успеха надпись → «Выключить», индикация «подключено».
  «Выключить» → корректно остановить поток/TX (SET_STREAM_MODE 0, TX_STOP при
  активности), RX_STOP, закрыть порт, надпись → «Включить».
- Кнопки Старт/Стоп/Пуск TX активны только при подключённом порте (иначе заблокированы).

## 3. Передатчик M1 (self-contained loopback для исследователя)

МКС: M1=TX (DW_TX_SOURCE_DEV), M2=RX. Дать исследователю управлять M1 из GUI —
контролируемый источник (loopback M1→M2), чтобы смотреть отклик канала на известный
сигнал без внешних передатчиков/китов.

### 3.1. Кнопки на инфо-панели
- **«Пуск TX»**: послать TX_PERIODIC с периодом и payload из Настроек (§3.2).
  Требует, чтобы модули были инициализированы (после Старт/INIT) и M1 живой; иначе
  RADIO_ERROR — показать в статусе.
- **«Стоп TX»**: послать TX_STOP.
- Индикация состояния TX (например, надпись/цвет кнопки: TX идёт / остановлен).

### 3.2. Настройки TX (вкладка Настройки)
- Поле «Период TX, мс» (дефолт напр. 20; минимум 5 — как в прошивке).
- Поле «Payload TX (hex)» (дефолт напр. `DE AD BE EF 01`).
- Эти значения использует кнопка «Пуск TX» и команда `txperiodic` в сценарии.

### 3.3. txperiodic в языке сценария
- Добавить в набор команд сценария `txperiodic <период> [payload hex...]` (txstop уже
  есть). Тогда исследователь может собрать self-contained эксперимент одной кнопкой
  Старт, напр.:
  ```
  init
  setphy 2 2 128 9 64 8     # Mode 4
  txperiodic 20 DE AD BE EF 01   # M1 передаёт
  rxstart
  stream 1
  ```
  → M1 шлёт, M2 принимает свой сигнал, поток/CIR/водопад показывают отклик канала.
- Валидация: период ≥5; payload — корректный hex; иначе ошибка сценария (стоп, не
  выполнять), как для прочих команд.

> Примечание: TX и RX на одной плате/ЦП конкурируют — для исследователя это ОК (умеренный
> темп, не 200 fps). Это осознанный self-contained режим, не замер пропускной.

## 4. Доводка водопада

### 4.1. Фиксированная глубина (главный фикс)
Сейчас водопад рисует ВЕСЬ буфер (растёт от старта до 500, «уплотняется книзу»).
Сделать фиксированную глубину отображения:
- Показывать только последние **N** кадров (N — константа по умолчанию, напр. 120,
  + поле «Глубина водопада» на вкладке Настройки, диапазон напр. 20..500).
- Высота водопада ПОСТОЯННАЯ: новые кадры сверху, старые (за пределом N) уходят снизу
  и исчезают — картинка «течёт» с постоянной высотой, а не растёт/уплотняется.
- Буфер deque(maxlen=500) оставить (глубина отображения ≤ буфера); imshow строить по
  последним N из буфера.

### 4.2. Поведение в тишине (нет сигнала)
- Договорено: «водопад шума без сигнала» — задача обнаружителя (сырой мониторинг
  аккумулятора), НЕ здесь.
- Здесь только: при отсутствии кадров (поток молчит) водопад НЕ растёт и НЕ ломается —
  показывает последнее накопленное ИЛИ заглушку «нет сигнала» (на выбор CC; проще —
  оставить последний кадр-буфер статичным, он и так не пополняется). Главное — не
  расти и не сыпать ошибок.

## 7. Проверка на железе (ВАГАН)
Self-contained loopback (без внешних источников): одна плата МКС (M1+M2), COM3.
1. Включить (COM3) с панели → подключено.
2. Настройки: режим Mode 4 (или 3), content=1, период TX 20, payload по умолчанию,
   глубина водопада 120.
3. Старт (сценарий init→setphy→rxstart→stream; TX ещё не идёт) → кружок серый (нет
   сигнала — M1 молчит), поток пуст.
4. Пуск TX (панель) → M1 передаёт → M2 принимает свой сигнал → кружок зелёный, CIR
   живой, телеметрия идёт. Loopback работает из GUI.
5. Вкладка Водопад (телеметрия видна сверху): течёт вниз, ПОСТОЯННАЯ высота (~120),
   старые уходят — не уплотняется. Сменить глубину в Настройках → высота меняется.
6. Стоп TX → кружок через ~1с серый, водопад/график перестают пополняться (не растут).
7. Телеметрия видна на всех вкладках; Выключить с панели на любой вкладке → корректный
   останов, порт закрыт.
8. Регресс: запись (Полный) metrics+cir; сценарий с txperiodic одной кнопкой Старт →
   loopback заводится целиком; закрытие/убийство GUI → консоль init OK.
9. Внешний источник (для сравнения): выключить M1 (Стоп TX), включить киты/плату2 →
   приём чужого сигнала так же работает.

Отчёт: работает ли loopback из GUI (Пуск/Стоп TX), фикс. высота водопада, телеметрия
на всех вкладках, Включить/Выключить с панели, регресс Шагов 1–3.
