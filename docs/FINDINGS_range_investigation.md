# FINDINGS — расследование дальности связи МКС (§15.1)

**Автор:** Vagan Sarukhanov. **Дата:** 2026-07-22. **Тип:** итоговый инженерный
документ (для инженеров-аппаратчиков). **Вывод:** программные средства исчерпаны
замерами — ограничение **аппаратное** (вероятно, модуль-клон + RF-разводка платы).

---

## 1. Симптом
- Линк **плата↔плата** (МКС): устойчиво **~2 м**, RSSI **−91…−94 dBm на 2 м**,
  случайные обрывы даже без препятствий.
- **Эталон — киты EVK1000** на том же столе, тот же режим (ch2 / Mode 3, настройки не
  менялись): **~8.5 м** прямой видимости, рвётся только за 2 поворотами/стенами.

## 2. Что проверено программно и ОПРОВЕРГНУТО замером
Каждый пункт: гипотеза → правка/проверка → результат на железе.

1. **Калибровка TX не выполнялась.** `dwt_configuretxrf()` (PG_DELAY + TX_POWER) звалась
   ТОЛЬКО ручной `SET_TX_POWER`; штатный 2-й шаг DecaWave (`dwt_configure()` **затем**
   `dwt_configuretxrf()`) выпадал → передача на сбросовом дефолте.
   → **Исправлено:** авто-калибровка в `SET_PHY_CONFIG`.
   → **Результат:** помогло **слабо** (RSSI −78→−82, дальность 1.5–2 м).
2. **Уровень TX-мощности.** Ползунок + пресеты в GUI-станции; значения сверены по
   **UM §7.2.31.4 Table 20 manual** (ch2/PRF64 «стандарт под маску» `0x67676767`,
   «железный макс» `0x20202020`; прежний дефолт `0x0E080222` был **ch5-SMART** —
   исправлен на канальный manual).
   → **Результат:** мощность **двигает RSSI**, но **не даёт стабильности/дальности**
   даже на «железном максе».
3. **Приёмные регистры (UM §2.5.5).** Проверены по коду: `dwt_configure()` выставляет
   **весь §2.5.5 верно под Mode 3** (AGC_TUNE1/2, DRX_TUNE0b/1a/1b/2, LDE, FS_PLL,
   RF_RXCTRLH); наш `dwt_config_t` корректен (ch2/110k/PLEN1024/PRF64/PAC32/code9).
   → **Результат:** ничего не «выпало» (в отличие от TX_POWER) — приёмная настройка верна.
4. **Горячий путь приёма.** Приёмник был выключен на каждом кадре на время `CaptureCIR`
   (~250 Б SPI, безусловно) + USB-стрима.
   → **Исправлено:** `CaptureCIR` только когда нужен (content=1 / командный режим),
   `dwt_rxenable` перевзводится раньше USB-стрима.
   → **Результат:** **не помогло** (потери кадров не от этого).
5. **Однокабельный полудуплекс (`MKS_SIMPLEX`).** Самый чистый тест гипотезы
   самоглушения: вся связь через ОДИН модуль M2, второй (M1) полностью заглушён
   (`forcetrxoff`, не конфигурится); TX/RX разнесены во времени (как EVK).
   → **Результат:** **гипотеза самоглушения ОПРОВЕРГНУТА** — дальность не улучшилась (~2 м).

## 3. Что доказано ИСПРАВНЫМ
**Приёмный тракт МКС исправен:** наша плата **ловит EVK на 5 м при −95 dBm с рабочим
CIR** (чёткий первый путь, физичные метрики). Т.е. приёмник слышит слабый далёкий
сигнал. **Асимметрия:** наш TX на 2 м даёт у своего же приёмника ≈ −94 dBm — столько же,
сколько EVK доставляет с 5 м → **наш излучатель/линк ≈ на ~8 dB слабее EVK** даже на
калиброванной мощности. Проблема на стороне **передачи/РЧ**, не приёма.

## 4. ВЫВОД: дефицит аппаратный
Все программные гипотезы проверены замерами и исчерпаны. Вероятный корень — **модуль**:

> МКС собран на **DWM1000-SMA rev.2.0 производства YCHIOT Ltd (Китай)** — **клон** со
> встроенной SMA-антенной, **НЕ оригинальный DecaWave DWM1000**. Его реальные
> характеристики/дальность/особенности **неизвестны**; мы обращались с ним как с
> оригиналом DecaWave, что **неверно**.

**Возможные причины (для аппаратной проверки):**
- более слабый TX-тракт модуля-клона (мощность/спектр/PA);
- неиспользуемые или иные особенности клона (регистры/OTP/калибровка кварца);
- ограничения согласования / RF-разводки платы МКС (импеданс, переходники, потери).

## 5. Передача инженерам-аппаратчикам (что делать дальше)
- Сравнить **передающий** (и, возможно, приёмный) тракт МКС с фирменным **EVK1000 /
  DW1000**: осциллограф / анализатор спектра (спектральная маска, мощность, PG-delay
  по факту), сравнение экземпляров модулей.
- Ревизия **RF-разводки** платы МКС (согласование, u.FL/SMA-переходники, длины/потери).
- Проверить **уход несущей / XTAL-trim** клона vs EVK (заводская подстройка <2 ppm у EVK).

## 6. Инженерный итог (стратегия)
- **МКС отлично подходит для ИССЛЕДОВАТЕЛЬСКИХ задач** (сбор метрик/CIR потоком,
  визуализация, MATLAB, контролируемый источник) — на близких дистанциях всё работает.
- **Двусторонний обмен текст/голос/видео на дальность** лучше строить на **другой
  платформе:**
  - более мощный МК (возможно, с аудиокодеком);
  - **оригинальные DecaWave DWM1000** или новые **Qorvo DWM3000** вместо клона YCHIOT;
  - опционально **микро-ПК (Linux)** вместо ноутбука на конце.
- Это к обсуждению, не в текущей работе (см. §15 HANDOFF «дальнейшие платформы»).
