# TASK_multimodule_detect — динамическое обнаружение живых модулей (INIT) + GET_STATUS

**Проект:** МКС / прошивка STM32F411 + DWM1000
**Автор задания:** Vagan Sarukhanov
**Исполнитель:** Claude Code (CC)
**Дата:** 2026-07-17
**Тип:** правка прошивки (protocol.c) + правка PROTOCOL_SPEC.md.
**Основание:** новая плата МКС с распаянным только M2 даёт INIT → RADIO_ERROR, т.к.
цикл INIT падает на отсутствующем M1 и портит общий статус. Нужно: INIT обнаруживает
живые модули (по dev_id), работает с живыми, не падает целиком из-за отсутствующего.
Масштабируется на будущий 5-канальный макет-обнаружитель.

## 2. Факты из кода (подтверждены, использовать дословно)

- `DWT_DEVICE_ID = 0xDECA0130` (deca_device_api.h:73). Использовать ЭТОТ макрос, НЕ
  хардкодить число.
- `uint32 dwt_readdevid(void)` (deca_device_api.h:330) — читает рег. DEV_ID (0x00) по
  SPI. Vendor сам применяет паттерн `dwt_readdevid() != DWT_DEVICE_ID` как проверку
  присутствия чипа (deca_device.c:1990,2004) — наш критерий «живой» совпадает с этим.
- `deca_port_select_device(i)` — выбирает шину/CS и переключает локальные данные
  драйвера. Возвращает DWT_SUCCESS/ERROR. Для i в [0..DW_DEVICE_COUNT-1] валиден
  (DWT_NUM_DW_DEV=2 задан в компиляторе — массив на 2 чипа).
- `dw_dev_state[i]` уже имеет поля `initialized` (u8) и `dev_id` (u32). Команды
  (RX_START/TX_*/SET_PHY/GET_SIGNAL_METRICS) УЖЕ проверяют `.initialized` конкретной
  роли перед работой — т.е. механизм «работать только с готовым модулем» уже есть,
  надо лишь корректно проставлять `.initialized` и не ронять общий статус INIT.
- Текущий INIT (protocol.c ~строки 391-422): цикл по DW_DEVICE_COUNT; при любой
  ошибке select/initialise → `status = RADIO_ERROR; continue;`. Именно это роняет
  общий статус из-за отсутствующего M1.
- Роли: `DW_TX_SOURCE_DEV = DW_DEV_M1` (0), `DW_RX_LISTEN_DEV = DW_DEV_M2` (1)
  (board_config.h). **Роли оставляем СТАТИЧНЫМИ** — живость решает, работает ли роль.

## 3. Прошивка — новый INIT (обнаружение живых)

Переписать `HandleINIT` так:

Для каждого места `i` в [0 .. DW_DEVICE_COUNT-1]:
1. `dw_dev_state[i].initialized = 0; dw_dev_state[i].dev_id = 0;`
2. `if (deca_port_select_device(i) != DWT_SUCCESS) { /* место недоступно */ continue; }`
   (НЕ ставить общий RADIO_ERROR — просто модуль не живой.)
3. `deca_port_hard_reset(i); deca_port_spi_set_slow();`
4. Прочитать dev_id ДО тяжёлой инициализации: `uint32 id = dwt_readdevid();`
   - Если `id != DWT_DEVICE_ID` → модуля нет/не отвечает → `continue` (не живой).
     Это отсекает отсутствующий M1 быстро, без ложного dwt_initialise.
5. Модуль присутствует. Полная инициализация: `if (dwt_initialise(DWT_LOADUCODE)
   != DWT_SUCCESS) { continue; }` (присутствует, но init не удался — не помечаем
   живым; редкий случай).
6. Успех: `dw_dev_state[i].initialized = 1; dw_dev_state[i].dev_id = id;`
   Инкремент счётчика живых.

После цикла:
- Посчитать `live_count` (сколько `initialized==1`) и `live_mask` (бит i = модуль i
  живой; бит0=M1, бит1=M2).
- **Возврат:** если `live_count == 0` → `STATUS_RADIO_ERROR` (нечем работать).
  Иначе → `STATUS_OK`, а в DATA положить:
  ```
  DATA (2 байта): live_count u8, live_mask u8
  ```
  (`*out_data` → статический буфер, `*out_len = 2`.)

> Порядок 4-перед-5 важен: сначала дешёвая проверка dev_id (быстро отсекает
> отсутствующий модуль), только для присутствующих — тяжёлый dwt_initialise с
> загрузкой микрокода. Это и ускоряет INIT на неполной плате (не ждём таймаутов
> initialise на мёртвом M1), и делает критерий «живой» точным (dev_id, как в vendor).

> Мёртвый модуль: `dwt_readdevid()` на отсутствующем M1 (SPI2 без ведомого) вернёт
> НЕ 0xDECA0130 (мусор/0x0/0xFFFFFFFF или таймаут в readfromspi → мусор). В любом
> случае `!= DWT_DEVICE_ID` → корректно отсекается. Если readfromspi по таймауту
> оставит мусор — это ок, важно лишь, что id != DEVICE_ID.

> На будущее (5 модулей): логика уже обобщённая — цикл по DW_DEVICE_COUNT, счётчик
> и маска. При DW_DEVICE_COUNT=5 заработает без изменений. Реализовать под текущий
> =2, не переусложняя, но код НЕ завязывать жёстко на «ровно 2».

## 4. Команды при неполной комплектации (проверить, менять НЕ требуется)

Роли статичны (§2). Ожидаемое поведение (должно получиться АВТОМАТИЧЕСКИ, т.к.
команды уже смотрят `.initialized`):
- Плата с одним M2 (RX): `RX_START`/`SET_PHY`/`GET_SIGNAL_METRICS`/`GET_CIR`/
  `SET_STREAM_MODE` — на DW_RX_LISTEN_DEV=M2 (живой) → работают.
- `TX_FRAME`/`TX_PERIODIC`/`TX_STOP`/`SET_TX_POWER` — на DW_TX_SOURCE_DEV=M1
  (не живой) → возвращают `RADIO_ERROR` (проверка `.initialized`). Это правильно:
  на приёмной плате передавать нечем.

CC: НЕ добавлять авто-переназначение ролей. Просто УБЕДИТЬСЯ, что все командные
обработчики опираются на `dw_dev_state[<роль>].initialized` (уже так) — и при мёртвой
роли корректно вернут RADIO_ERROR, а не полезут в неинициализированный модуль.
Если найдётся обработчик, который НЕ проверяет `.initialized` перед доступом к своей
роли — добавить проверку (по образцу существующих). НЕ менять логику для живых ролей.

## 5. GET_STATUS — расширить живостью модулей

Сейчас GET_STATUS отдаёт 7 байт по M1 (TX_state, RX_state, channel, data_rate,
preamble_length u16, PRF). **Не ломать существующие 7 байт** (хост их парсит).
Добавить В КОНЕЦ новые поля:
```
... существующие 7 байт ...
+ live_count u8
+ live_mask  u8
+ dev_id[0]  u32 LE   (M1; 0 если не живой)
+ dev_id[1]  u32 LE   (M2; 0 если не живой)
```
Итого новый размер = 7 + 1 + 1 + 4 + 4 = 17 байт. Хост, читающий старые 7 байт,
не сломается (просто прочитает первые 7); новый хост/консоль сможет прочитать
расширение. В SPEC описать новый формат и отметить, что первые 7 байт — прежние.

> Если проще и чище — можно ввести верс* поля, но не обязательно; главное: первые
> 7 байт байт-в-байт прежние, расширение — в конце.

## 6. Проверка на железе (ВАГАН, после ШАГА 0) — процедура

На ОДНОМОДУЛЬНОЙ плате (только M2), порт этой платы (напр. COM4):
1. Перепрошить свежим билдом BOARD_MKS (ШАГ 0).
2. `python mks_console.py COM4`
3. `init` → ожидаем: **STATUS = OK**, DATA = `01 02` (live_count=1, live_mask=0b10 —
   жив только M2, бит1). Если так — обнаружение работает, INIT больше не падает.
4. `status` → в расширении: live_count=1, live_mask=0x02, dev_id[0]=0 (M1 нет),
   dev_id[1]=0xDECA0130 (M2 живой). Это подтверждает: M2 отвечает, M1 отсутствует.
5. `mode3` → OK (SET_PHY на M2). `rxstart` → OK (RX на M2). Приём заработал.
6. `txframe DE AD BE EF 01` → ожидаем **RADIO_ERROR** (M1-передатчика нет) — это
   корректное поведение неполной платы.

На ДВУХМОДУЛЬНОЙ плате (регресс — ничего не сломали):
7. `init` → OK, DATA = `02 03` (live_count=2, live_mask=0b11 — оба живы).
8. Прежние сценарии (loopback M1→M2, метрики, CIR, поток) работают как раньше.

Отчёт: DATA от init на обеих платах, вывод status (live_count/mask/dev_id), что
приём на одномодульной пошёл, а передача честно отказала; двухмодульная не сломана.
