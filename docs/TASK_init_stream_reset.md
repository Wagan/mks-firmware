## Зачем
Если приложение (GUI/probe) включило поток `SET_STREAM_MODE 1` и закрылось АВАРИЙНО
(краш, убитый процесс, выдернутый шлейф) — плата остаётся в потоковом режиме и сыплет
потоковые кадры в USB. Следующее приложение (особенно консоль) тогда захлёбывается
чужим потоком и не может работать.

Фундаментальная страховка: раз любое приложение начинает работу с INIT, пусть INIT
принудительно выключает поток. Тогда что бы ни случилось с предыдущим приложением —
новое после `init` получает чистый канал (командный режим).

## Правка (protocol.c, HandleINIT)
В `HandleINIT` (там уже сбрасываются `cir_snap.valid`/`rx_metrics.valid` после
обнаружения модулей) добавить сброс потокового состояния:
```c
    stream_active = 0;     /* страховка: гарантированно выйти из потокового режима */
    /* при желании — и stream_seq/stream_dropped не трогаем: они сбрасываются при
       следующем включении потока (mode==1|2 в HandleSET_STREAM_MODE) */
```
> Свериться с фактическим именем флага в коде (`stream_active` — из CIR-2a).
> Сбрасывать ТОЛЬКО активность потока. `stream_content` можно не трогать (он
> применяется только при активном потоке). Счётчики seq/dropped не трогаем.

## Границы
- Только `stream_active = 0` в HandleINIT. Больше ничего.
- Не трогать: vendor, deca_port, board_config, командный протокол/CRC/парсер,
  другие обработчики, хост, GUI.

## Самопроверка (CC, без железа)
- Сборка Debug чистая. Имя флага сверено с кодом. INIT после правки: модули
  обнаружены + valid-флаги сброшены (как было) + stream_active=0 (новое).
- НЕ запускать на железе.

## Проверка на железе (Ваган, после ШАГА 0)
1. `python mks_stream_probe.py COM3 --mode 4 --content 1 --seconds 5` — включит поток.
   НЕ дожидаясь корректного стопа, можно просто закрыть/прервать (Ctrl+C посреди).
2. Сразу `python mks_console.py COM3` → `init` → должно пройти OK, и далее команды
   (`status`, `mode4`, `cir`) работают нормально — поток погашен на INIT, консоль
   не захлёбывается.
3. Если до правки консоль после прерванного probe «сыпалась»/не отвечала — теперь
   `init` очищает ситуацию.
