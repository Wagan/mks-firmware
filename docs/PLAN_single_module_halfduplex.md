# PLAN — однокабельная сборка: полудуплекс на одном модуле (§15.1 дальность)

**Дата:** 2026-07-22. **Основание:** RECON_single_module_halfduplex.md (`5e9399b`).
**Гипотеза:** M1 (TX) глушит M2 (RX) на одной плате. **Первый заход:** compile-time
однокабельная сборка — станция-полудуплекс на M2, M1 выключен. Проверить, прыгнет ли
дальность к EVK (8.5 м) с прежних ~2 м. Рантайм-переключение — потом.
**Статус: PLAN на утверждение ORDER, код НЕ трогаем.** Метки: Wagan + Dima.

## Рамка (архитектор)
Compile-time отдельная сборка (не рантайм); двухмодульную (M1 TX/M2 RX, CIR/loopback)
не трогаем. Рабочий модуль **M2** (совпадает с третьей платой). Реализовать связку
полудуплекса RX↔TX на одном чипе (сейчас нет). M1 заглушить (`dwt_forcetrxoff`, sleep
как запас). Прикладной слой станции/GUI — как есть.

## 0. M2 vs M1 — по коду разницы в сложности НЕТ
Роли — это индексы устройства (`deca_port_select_device(idx)`), путь симметричен для
любого индекса. **M2 предпочтителен** по причине третьей платы (только M2) — код станет
единым для одномодульной платы. Берём M2.

## 1. Как задаётся однокабельная сборка (compile-time)
`App\board\board_config.h`, секция `BOARD_MKS` — добавить под-флаг (сборку переключает
`-D MKS_SIMPLEX` в отдельной build-конфигурации CubeIDE, ЛИБО раскомментировать одну
строку `#define MKS_SIMPLEX`):
```c
/* #define MKS_SIMPLEX */   /* однокабельная сборка: полудуплекс на M2, M1 выключен */
...
#ifdef MKS_SIMPLEX
  #define DW_RX_LISTEN_DEV  DW_DEV_M2
  #define DW_TX_SOURCE_DEV  DW_DEV_M2      /* оба направления на ОДИН чип */
  #define DW_SINGLE_MODULE  1
#else
  #define DW_RX_LISTEN_DEV  DW_DEV_M2
  #define DW_TX_SOURCE_DEV  DW_DEV_M1      /* как сейчас (двухмодульный) */
  #define DW_SINGLE_MODULE  0
#endif
```
- **`DW_DEVICE_COUNT` остаётся 2** (M2 = индекс 1; INIT должен доходить до индекса 1).
- Пины MKS не меняются (та же плата). `DW_SINGLE_MODULE` — компайл-гейт нового кода.
- Двухмодульная сборка (`MKS_SIMPLEX` не задан) — **байт-в-байт как сейчас**
  (`DW_SINGLE_MODULE=0`, весь новый код под `#if` выпадает).

## 2. Точки ролей (§1 разведки) — сводятся на M2 БЕСПЛАТНО
Все ~13 точек уже используют `DW_TX_SOURCE_DEV`/`DW_RX_LISTEN_DEV` (не хардкод индексов).
В simplex оба `#define` = M2 → `deca_port_select_device` везде выбирает M2. **Правок в
самих точках не требуется** — редирект ролей даёт `#define`. Меняется только логика
полудуплекса (§3) и заглушение M1 (§4), обе под `#if DW_SINGLE_MODULE`.

## 3. Связка полудуплекса RX↔TX на одном чипе (ГЛАВНОЕ, под `#if DW_SINGLE_MODULE`)
Main loop однопоточный: `PollRx → PollRadio → PollTx` (`main.c:120–122`), последовательно
→ **параллельной гонки за чип НЕТ**. Задача — корректная секвенция TX и ОБЯЗАТЕЛЬНЫЙ
re-enable RX после TX (иначе после передачи чип останется глух: `PollRadio` включает RX
только в ветке RXFCG, «на пустом месте» не перевзводит).

**Где вставить (3 TX-точки):**
- **`HandleTX_FRAME`** (`:1051`+): перед `dwt_starttx` — `#if DW_SINGLE_MODULE
  dwt_forcetrxoff();` (снять RX). После busy-wait TXFRS и снятия флага — `#if
  DW_SINGLE_MODULE dwt_rxenable(DWT_START_RX_IMMEDIATE);` (вернуть RX).
- **`PROTOCOL_PollTx`** (`:1373`+, периодика — beacon): та же обвязка forcetrxoff→
  starttx→wait TXFRS→rxenable (весь блок уже per-frame; добавить forcetrxoff в начале и
  rxenable в конце под `#if DW_SINGLE_MODULE`).
- **`HandleTX_STOP`** (`:1084`, уже делает `dwt_forcetrxoff`): в simplex после
  forcetrxoff добавить `#if DW_SINGLE_MODULE dwt_rxenable(...)` — чтобы после останова
  передачи чип продолжил слушать (в двухмодульном RX — другой чип, не трогаем).

**Координация PollTx/PollRadio:** т.к. main loop последователен, TX-операция
(forcetrxoff→starttx→busy-wait→rxenable) **атомарна** (main loop заблокирован в
busy-wait, `PollRadio` не вклинится). Кадр, пришедший В МОМЕНТ передачи, теряется
(forcetrxoff обрывает приём) — это неотъемлемо для полудуплекса, приемлемо.

**Двухмодульный (`DW_SINGLE_MODULE=0`):** весь этот код под `#if` — не компилируется,
поведение прежнее (TX на M1 без forcetrxoff/rxenable, RX на M2 непрерывный).

## 4. Заглушение M1 (под `#if DW_SINGLE_MODULE`)
- В `HandleINIT` (`:447–493`), после цикла обнаружения живых, добавить: для каждого
  инициализированного модуля `i != DW_RX_LISTEN_DEV` → `deca_port_select_device(i)` +
  `dwt_forcetrxoff()` (M1 в IDLE, не передаёт). На третьей плате (M1 отсутствует) —
  модуль не инициализирован → пропускается.
- **Подрешение (для чистоты наводки):** `HandleSET_PHY_CONFIG` (`:713`+) в двухмодульном
  конфигурит ВСЕ живые (включая M1) → снова поднимает PLL M1. В simplex **конфигурить
  только `DW_RX_LISTEN_DEV`** (пропускать прочие), чтобы M1 остался неактивным. Иначе
  `forcetrxoff` из INIT перекроется конфигом. → развилка ORDER (B).
- **Запас (если forcetrxoff мало — PLL/клок M1 ещё наводят):** `dwt_entersleep()` на M1
  (глубже, PLL off), требует `dwt_configuresleep()` до и `dwt_spicswakeup()`/CS для
  выхода. Для первого захода — `forcetrxoff`; `entersleep` — вторым шагом, если замер
  покажет остаточную наводку.

## 5. Что с loopback
- В simplex **self-loopback НЕВОЗМОЖЕН** (один чип не принимает собственную передачу
  одновременно). Проверка на столе — **две однокабельные платы** (две станции) ЛИБО одна
  simplex-станция против **EVK**. Двухмодульная сборка сохраняет loopback (M1→M2) для
  CIR/метрик/исследований.

## 6. Одномодульная совместимость (третья плата, только M2) — барьер §5 разведки снят
- `HandleINIT` переживает 1 живой (`RADIO_ERROR` только при 0). На плате только с M2:
  live_count=1, INIT OK.
- В simplex TX-роль = M2 (не M1) → `HandleTX_FRAME`/`PollTx` проверяют
  `dw_dev_state[M2].initialized` (жив) → **передача работает**. Барьер «TX прибит к
  отсутствующему M1» снят. Тот же бинарник — и для одномодульной платы.

## 7. Риски
- **Гл. риск (по факту НЕ гонка):** main loop последователен → PollTx/PollRadio не
  сталкиваются параллельно. Реальный риск — **RX остался выключен после TX**, если
  забыть `rxenable` в TX-пути (§3). Митигация: rxenable во ВСЕХ трёх TX-точках (TX_FRAME/
  PollTx/TX_STOP) под `#if DW_SINGLE_MODULE`.
- **Потеря кадра во время передачи** — неотъемлемо (полудуплекс), приемлемо.
- **Наводка M1 через PLL** несмотря на forcetrxoff — митигация: не конфигурить M1 (§4
  подрешение) + `entersleep` запасом.
- **Двухмодульную не сломать** — весь новый код под `#if DW_SINGLE_MODULE`; при
  `MKS_SIMPLEX` не заданном — прежнее поведение.
- **`DW_DEVICE_COUNT` держать 2** — иначе INIT не дойдёт до M2 (индекс 1).
