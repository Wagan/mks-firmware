# Авто-калибровка TX-тракта под канал при SET_PHY_CONFIG (§15.1 дальность)

**Дата:** 2026-07-22. **Основание:** RECON_tx_power_range.md (`394b6c5`) — плохая
дальность (2–3 м, человек рвёт связь), RSSI ~−78 dBm вблизи; антенный тракт исключён.
Причина: `TX_POWER`/`PG_DELAY` у станции в **сбросовом дефолте** — `dwt_configuretxrf()`
зовётся только ручной `SET_TX_POWER`, которую станция не вызывает. **Правка прошивки.**

Разбито: PLAN-набросок (ORDER согласован — вариант A) → правка прошивки → доки.

---

## Рамка архитектора (соблюдена)
- Калибровка TX под канал — **автоматически при `SET_PHY_CONFIG`** (не только по ручной
  `SET_TX_POWER`), т.к. PG_DELAY + канальная TX_POWER — неотъемлемая часть настройки
  канала (штатный поток DecaWave: `dwt_configure()` **затем** `dwt_configuretxrf()`).
- Дефолт: `PGdelay = map_pgdelay(channel)` (ch2 = `0xC2`), `power = TX_POWER_MAN_DEFAULT`
  (`0x0E080222`, `deca_regs.h:534`), `dwt_setsmarttxpower(0)` (manual, DIS_STXP=1).
- `SET_TX_POWER` (0x11) не тронут — ручная подстройка поверх дефолта, его `power`
  перекрывает `0x0E080222`.
- Чинит все сценарии (станция/GUI/probe) — правок в `tools\` не требуется.

## ORDER (согласован — вариант A)
Врезка **внутри существующего цикла** по живым модулям в `HandleSET_PHY_CONFIG`, сразу
после `dwt_configure(&cfg)` (устройство уже выбрано `deca_port_select_device(i)`):
```c
dwt_configure(&cfg);
uint8_t pgdly;
if (map_pgdelay(channel, &pgdly)) {          // валидный канал -> всегда true
    dwt_txconfig_t txcfg;
    txcfg.PGdly = pgdly;                      // ch2 = 0xC2
    txcfg.power = TX_POWER_MAN_DEFAULT;       // 0x0E080222
    dwt_setsmarttxpower(0);                   // manual, per-module
    dwt_configuretxrf(&txcfg);               // пишет PG_DELAY + TX_POWER
}
dw_dev_state[i].channel = channel; ...
```
- **Per-module** (не отдельным вторым проходом) — устройство уже выбрано, лишнего
  `deca_port_select_device` нет; `dwt_setsmarttxpower(0)` тоже per-module (пишет `SYS_CFG`
  выбранного устройства).
- **Ошибки:** `dwt_configuretxrf` — `void`. Единственная развилка `map_pgdelay`; для
  валидного канала (уже прошёл `map_channel`, тот же набор `{1,2,3,4,5,7}`) всегда `true`.
  При `false` (не должно) — TX-калибровку **пропускаем, `SET_PHY_CONFIG` не роняем**.

## Что реально изменилось на проводе/в эфире
- **До:** `SET_PHY_CONFIG` → `dwt_configure` (RF_TXCTRL по каналу ✓, но TX_POWER/PG_DELAY
  — сбросовый дефолт). Передача на заниженной, неоткалиброванной под ch2 мощности.
- **После:** `SET_PHY_CONFIG` → `dwt_configure` + `dwt_configuretxrf` (PG_DELAY 0xC2,
  TX_POWER 0x0E080222, manual). Передатчик откалиброван под ch2.
- **SET_TX_POWER** по-прежнему может поднять/опустить уровень поверх дефолта (level →
  `0xFF−level` во все октеты `TX_POWER`).

## Приёмка на железе (ВАГАН) — КЛЮЧЕВОЙ критерий: замер дальности до/после
1. Прошить новую прошивку. Сценарий как обычно: `init → set_phy(Mode3) → …` (калибровка
   TX применяется теперь автоматически внутри `set_phy`).
2. **Замер дальности** двумя станциями (или GUI-станцией) — сравнить с прежними 2–3 м:
   ожидается **рост устойчивой дальности** и **рост RSSI на той же дистанции** (вблизи
   было ~−78 dBm — должно подрасти).
3. Косвенно: `metrics`/GUI-шкала — RSSI выше при том же расположении.
4. Регресс: `SET_TX_POWER` (ручной `txpower <level>` в консоли) по-прежнему меняет
   уровень (проверить, что перекрывает дефолт — RSSI растёт с level).
5. Если дальность НЕ выросла — значит TX-мощность не единственный фактор (вернуться к
   прочим кандидатам: PHY-режим/скорость, чувствительность приёма, среда). Но по коду
   это был подтверждённый недокрут (TX-тракт на сбросе).

**Отчёт Вагана:** дальность до/после; RSSI на фикс. дистанции до/после; работает ли
ещё ручной `txpower` поверх; не сломался ли приём/чат.

## Дальше (§15.1)
Если дальность выросла достаточно — двигаться к этапу (2) надёжного канала (ARQ).
При желании — подстроить дефолтный `power` (сейчас vendor `0x0E080222`) под регуляторный
потолок ch2 замером спектра/EIRP (отдельный заход).
