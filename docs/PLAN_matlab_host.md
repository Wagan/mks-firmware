## 1. Цель и объём

**Цель:** MATLAB-библиотека, дающая тот же функционал, что `tools/mks_protocol.py`:
кадрирование + CRC8, команды, разбор ответов (метрики/CIR) в MATLAB-структуры/
таблицы, и **потоковый (непрерывный) сбор** данных в `table`/массивы для анализа.

**Входит:**
- Слой протокола на MATLAB (SYNC/LEN/CMD/CRC8, отправка, приём с поиском SYNC + таймаут).
- Методы команд: INIT, SET_PHY_CONFIG, RX_START/STOP, GET_SIGNAL_METRICS, GET_CIR
  (и мелочь PING/GET_STATUS/TX* по образцу — по необходимости).
- Парсеры: метрики (18/28/30 байт по длине) и CIR (заголовок + I/Q) → struct/table.
- **Потоковый сбор**: непрерывный цикл опроса (как фоновый поток Python-GUI), сбор
  N кадров / за T секунд, с фильтром уникальных кадров (рост `count`) → `table`.
- Экспорт: `table` → `writetable`/CSV и/или `.mat`.
- Примеры-скрипты для исследователя.

**НЕ входит (границы):**
- Изменения прошивки/протокола на проводе.
- Прошивочный **потоковый режим** (`SET_STREAM_MODE`, CIR-2) — отдельный будущий
  шаг; MATLAB-хост v1 работает по существующей модели «команда→ответ» в цикле.
- GUI на MATLAB, продуктовый детектор (не цель МКС, см. §1 handoff).

## 2. Опора (источники истины — НЕ по памяти)

- **Поведение протокола:** `tools/mks_protocol.py` (эталон: кадр, CRC8, команды,
  `parse_signal_metrics`, `parse_cir`). MATLAB-код должен давать байт-в-байт те же
  кадры и разбор.
- **Форматы на проводе:** `docs/PROTOCOL_SPEC.md` (§5 команды, §6 параметры,
  §8 форматы ответов: метрики 18/28/30 байт, GET_CIR заголовок 6 байт + I/Q).
- **CRC8 на MATLAB — уже есть эталон:** PROJECT_HANDOFF §12 (poly 0x07, init 0x00,
  без рефлексии/xorout, по LEN+CMD+PARAMS без SYNC). Переиспользовать, не изобретать.

## 3. Раскладка файлов (предложение)

Новая папка `matlab/` (рядом с `tools/`), пакет-класс:
```
matlab/
  +mks/                     % namespace-пакет MATLAB
    MKS.m                   % класс: открытие порта, команды, приём
    crc8.m                  % CRC8 (из handoff §12)
    parseSignalMetrics.m    % 18/28/30 байт → struct
    parseCir.m              % заголовок + I/Q → struct (fp_index, samples, amps)
    acquire.m               % потоковый сбор N кадров/T сек → table
  examples/
    example_connect.m       % init → mode3 → rxstart → metrics/cir
    example_stream.m        % непрерывный сбор в table + сохранение
  README.md                 % запуск, требования (MATLAB R2019b+ serialport)
```
Финальную раскладку/имена согласуем в задаче (Ваган). Ничего в `tools/` не трогаем.

## 4. Слой протокола на MATLAB

- **Порт:** `serialport(port, 115200)` (R2019b+; для старых MATLAB — `serial`,
  вынести в открытый вопрос §10). `write(s, bytes, "uint8")`, `read(s, n, "uint8")`.
- **Сборка кадра** (как `build_command`): `LEN=1+numel(params)`;
  `body=[LEN CMD params]`; кадр = `[0xAA 0x55 body crc8(body)]`.
- **Приём ответа** (как `read_response`): искать SYNC `AA 55` в потоке, затем
  `LEN`, `LEN` байт тела, `CRC`; проверить `crc8([LEN body])`. Таймаут на команду
  (INIT — длиннее, ~20 с). Вернуть `[status, data]`.
- Все многобайтовые — little-endian (`typecast`/`bitshift`).

## 5. Команды (методы MKS)

По образцу Python (маппинг CMD из PROTOCOL_SPEC §5):
`ping`, `init` (long timeout), `getStatus`, `setPhyConfig(ch,dr,plen,code,prf,pac)`
(байты как в `cmd_setphy`), `rxStart`, `rxStop`, `getSignalMetrics`,
`getCir(half)`. TX-команды — по необходимости (для loopback-самопроверки).
`setPhyConfig` — те же 7 байт wire, что и в консоли/GUI.

## 6. Парсеры ответов → MATLAB

- **parseSignalMetrics(data):** по длине 18/28/30 (как Python). Вернуть struct с
  полями count, CIR_PWR, RXPACC, STD_NOISE, FP_AMPL1..3, FP_INDEX, MAX_NOISE; для
  финала — rxpacc_nosat, N_corrected, rssi_dbm, fp_power_dbm, snr_db + флаги
  валидности (INT16_MIN=-32768 → NaN). i16 через `typecast(uint16→int16)`.
- **parseCir(data):** заголовок fp_index/start_index/count (u16 LE) + `count` пар
  I/Q (int16 LE). Вернуть struct: fp_index, start_index, count, samples (Nx2 int16),
  amps (Nx1 double = hypot(I,Q)).

## 7. Потоковый сбор (ГЛАВНОЕ этого захода)

MATLAB де-факто однопоточный для пользовательского кода. Два подхода:

- **Вариант A (реком.): блокирующий цикл сбора.** Функция `acquire(dev, opts)`:
  крутит `getSignalMetrics` (+ опц. `getCir`) в цикле, пишет строку на **уникальный
  кадр** (рост `count`), пока не набрано `N` кадров или не вышло `T` секунд; возвращает
  `table` (столбцы = как metrics.csv из GUI: frame_id, timestamp, count, RXPACC,
  RXPACC_NOSAT, N_corrected, CIR_PWR, STD_NOISE, FP_INDEX, RSSI_dBm, FP_POWER_dBm,
  SNR_dB, mode). Плюс опц. отдельный вывод CIR (cell/структ-массив, связь по frame_id).
  Прозрачно, детерминированно, легко анализировать. Прерывание — Ctrl+C.
- **Вариант B (опц., позже): фоновый по `configureCallback`/`timer`.** Неблокирующий
  сбор через колбэк `serialport`. Сложнее и хрупче; для v1 не обязателен.

**Рекомендация:** Вариант A. «Потоковость» здесь = непрерывный сбор серии кадров в
таблицу (для длинных статистических прогонов и для прицельной формы CIR), поверх
существующей модели команда→ответ. Прошивочный поток (CIR-2) — отдельно, позже;
когда появится, `acquire` переключится на чтение потоковых кадров без смены API.

## 8. Экспорт под анализ

`table` → `writetable(t, 'run.csv')` (тот же дружественный формат, что GUI-CSV) и/или
`save('run.mat','t','cir')`. Формат столбцов держать идентичным `metrics.csv` GUI,
чтобы существующие MATLAB-скрипты читали и то, и то.

## 9. Примеры для исследователя

- `example_connect.m`: открыть COM → init → setPhyConfig(Mode 3) → rxStart → в цикле
  печатать RSSI/SNR/FP; getCir → plot формы CIR с маркером FP.
- `example_stream.m`: `t = mks.acquire(dev, struct('seconds',10,'cir',false))` →
  `writetable(t,'run.csv')`; гистограмма SNR/RSSI.

## 10. Открытые вопросы (для задачи)

1. **Версия MATLAB / API порта:** ориентируемся на `serialport` (R2019b+) или нужен
   fallback на старый `serial`? (Уточнить у исследователей.)
2. **Уникальность кадров:** фильтр по росту `count` (как GUI) — подтвердить.
3. **Частота опроса** в `acquire` (пауза цикла) и критерии остановки (N кадров / T сек / оба).
4. **Half для CIR** в потоковом сборе (0=дефолт прошивки/30) и писать ли CIR всегда
   или по флагу (объём: полный CIR-режим тяжёлый, как в GUI).
5. **Раскладка/имена** файлов пакета (§3) — финализировать.

## 11. Самопроверка (СС, без железа)

- Синтаксис MATLAB СС локально не исполнит (нет MATLAB в окружении) — проверяю
  логикой/сверкой с `mks_protocol.py` побайтно; при возможности прогоню CRC8/парсеры
  на эквиваленте (Octave, если доступен) или оставлю юнит-примеры в комментариях с
  ожидаемыми значениями (кадр PING, разбор тест-DATA метрик/CIR).
- Байтовые ожидания брать из Python-эталона (напр. кадр `mode3`, разбор 30-байт метрик).
