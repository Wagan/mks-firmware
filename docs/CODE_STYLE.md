# CODE_STYLE.md — стиль и шапки файлов

## Шапка-«рамка» для самодельных файлов

Во ВСЕ наши файлы исходного кода в `App/**` (`.c` и `.h`) добавлять шапку по
шаблону ниже. Поля `Файл:` и `Описание:` заполнять под конкретный файл.
**Номер версии в шапку НЕ писать** — он живёт в `App/version.h` (чтобы не
расходился по файлам).

```c
/*******************************************************************************
 *  МКС — Модуль коммуникации и сопряжения
 *  Прошивка для STM32F411 + 2x DWM1000 (DW1000)
 *
 *  Файл:     <имя_файла.c/.h>
 *  Описание: <краткое назначение модуля в одну-две строки>
 *
 *  Copyright (c) 2026 NCPR, Flexlab LLC. Все права защищены.
 *******************************************************************************/
```

## Правила

1. Шапку добавлять ТОЛЬКО в `App/**` (наш код).
2. **НЕ добавлять** шапку и НЕ модифицировать файлы в `Drivers/decadriver/` —
   это код DecaWave с их лицензией. Копирайт NCPR/Flexlab туда ставить нельзя.
3. Не трогать сгенерированные CubeMX файлы вне секций `USER CODE` (Core/**,
   Drivers/STM32F4xx_HAL_Driver/**, USB_DEVICE/** и т.п.).
4. Версия прошивки — только `FW_VERSION_STR` из `App/version.h`. Если где-то
   нужен номер версии в коде — включать `version.h`, не хардкодить.
5. Язык комментариев — русский (как в существующих файлах). Код/идентификаторы —
   латиница.
6. Стиль — C, STM32 HAL. Отступы 4 пробела, без табов.
