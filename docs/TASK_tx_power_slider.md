# TX-мощность: Table 20 manual + ползунок в GUI-станции (§15.1 дальность)

**Дата:** 2026-07-22. **Основание:** авто-калибровка TX (`TASK_tx_calibration.md`)
помогла слабо (RSSI −78→−82, дальность 1.5–2 м). Причина: дефолт `0x0E080222` — по UM
§2.5.5.6 это **ch5 SMART**, для ch2/manual неверный. **Значения сверены по UM §7.2.31.4
Table 20 (manual, DIS_STXP=1) — копия Вагана.** Зона Sergey/Wagan.

Разбивка (утв. ORDER: A — две кнопки, «Стандарт»=дефолт; B — guard-пропуск без ch2-подстановки):
прошивка → хост/GUI → доки.

---

## Выверенные значения — UM §7.2.31.4 Table 20 (manual, одноктетные)
`TX_POWER` регистр (0x1E), октеты одинаковы → выразимы через `level = 0xFF − октет`:

| Канал | PRF64 | PRF16 |
|---|---|---|
| ch1/ch2 | `0x67676767` | `0x75757575` |
| ch3 | `0x8B8B8B8B` | `0x6F6F6F6F` |
| ch4 | `0x9A9A9A9A` | `0x5F5F5F5F` |
| ch5 | `0x85858585` | `0x48484848` |
| ch7 | `0xD1D1D1D1` | `0x92929292` |

- **ch2/PRF64 «под маску»** (Mode 3): октет `0x67` → **level `0x98`**, регистр `0x67676767`.
- **Железный макс:** октет `0x20` → **level `0xDF`**, регистр `0x20202020` (~9 dB выше
  маски, лабораторный).
- Прежний `0x0E080222` — ch5 SMART (UM §2.5.5.6), для ch2/manual неверный.

## Шаг 1 — прошивка (`App\protocol\protocol.c`, коммит)
- **`tx_power_manual(channel, prf)`** — map по Table 20 (обе PRF). Канал вне {1..5,7} →
  `false` (guard-пропуск, симметрично `map_pgdelay`; ch2 НЕ подставляем).
- **`SET_PHY_CONFIG`:** канальный дефолт `TX_POWER = tx_power_manual(channel, prf_raw)`
  вместо `0x0E080222` (ch2/PRF64 → `0x67676767`). PGdelay/`setsmarttxpower(0)` как были;
  guard на оба map-а (при `false` TX-калибровку пропускаем, конфиг не роняем).
- **`SET_TX_POWER` (0x11):** + сырой 4-байтовый формат (`params_len==4` → `u32 LE` как
  есть, без клампа); level-формат (`params_len==1`) без изменений; иначе `INVALID_PARAM`.
- Синтакс-чек `-Wall -Wextra` — чисто.

## Шаг 2 — хост/GUI (коммит)
- `mks_protocol.set_tx_power_raw(reg_u32)` (4-байт формат); `set_tx_power(level)` как был.
- `mks_station`: зеркало `TX_STD_OCTET` + `std_tx_level(ch,prf)`/`level_to_reg`;
  `Station.cmd_q` + `request_tx_power(level)` (кламп 0..0xDF) + `_apply_tx_power`
  (исполнение ТОЛЬКО в engine-треде — владелец порта, между `poll`'ами), состояние
  `tx_level/tx_reg`, событие `txpower`. Консоль: `/power <level>`.
- `mks_station_gui`: панель **«Мощность TX (свой M1)»** — ползунок `level 0..0xDF`
  (вправо = мощнее, отправка по отпусканию мыши, не флудим), подпись
  `level=0xNN рег=0xYYYYYYYY [зона]` (под/выше маски); пресеты:
  - **«Стандарт (под маску) = дефолт set_phy»** → level из Table 20 по каналу (ch2 → `0x98`);
  - **«Железный макс (лаб., ~9 dB выше маски)»** → `0xDF`.
  Индикатор применённого уровня (по событию `txpower`).
- Применяется на **свою** станцию (свой M1), на обоих концах независимо.

**Юниты:** `level_to_reg` (0x98→0x67676767, 0xDF→0x20202020), `std_tx_level` по каналу/
PRF, зеркало==прошивочная Table 20, `cmd_q` кламп, `_apply_tx_power` событие; GUI smoke
(старт 0x98 для ch2, пресет 0xDF→cmd_q, зона, «применено»).

## Шаг 3 — доки (этот коммит)
SPEC §6 (калибровка = Table 20 + таблица значений + level-соответствие), §8
(`SET_TX_POWER` 4-байтовый формат); HANDOFF §15.1 (правки 1/2); этот отчёт.

---

## Порядок применения
Стартовая `open_and_setup` (init→**set_phy**→rx_start→stream) ставит **канальный дефолт
Table 20** (ch2 → `0x67676767` = level `0x98`). Ползунок/пресет шлёт `SET_TX_POWER`
**после** старта (клик всегда позже) → перекрывает дефолт. `set_phy` повторно не
зовётся → уровень держится. «Стандарт» возвращает к дефолту.

## Приёмка на железе (ВАГАН) — замер дальности
1. Прошить. Две станции/GUI (Mode 3, ch2/PRF64).
2. На передающей стороне: **«Стандарт (под маску)» (level 0x98, рег 0x67676767)** →
   замер дальности + RSSI у партнёра на фикс. дистанции. Ожидание: заметно лучше
   прежних 1.5–2 м (правильный ch2-manual вместо ch5-smart).
3. Затем **«Железный макс» (0xDF, рег 0x20202020)** → предельная дальность стенда;
   RSSI у партнёра на той же дистанции — должен подрасти ещё (~9 dB выше маски).
4. Ползунок между — RSSI/дальность монотонно растут с level.
5. Регресс: консольный `txpower <level>` / `/power` работают; сырой 4-байт (при желании)
   пишет регистр как есть; чат/приём целы.

**Отчёт Вагана:** дальность и RSSI на «Стандарт» и «Железный макс» (до/после, фикс.
дистанция); монотонность ползунка; не сломан ли чат/приём.

## Дальше (§15.1)
Если дальность практична — этап (2) надёжного канала (ARQ). При выходе за маску на
«железном максе» — помнить: это лабораторный режим, не для эксплуатации.
