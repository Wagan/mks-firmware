# decawave-examples — референс-код DecaWave (справочный)

Официальные примеры из `dwm1001-examples`, из которых мы взяли DecaDriver 4.0.6.
**Назначение — справочный материал** (образец правильной последовательности
вызовов драйвера). НЕ включать в сборку прошивки. Платформа примеров — Nordic
nRF52, но **логика работы с DW1000 идентична** нашей (те же `dwt_*` вызовы);
отличается только платформенный слой (SPI/GPIO/задержки).

## Файлы

- `ss_resp_main.c` — **responder** (приёмник) single-sided two-way ranging.
  Ключевой референс для нашего RX-тракта: показывает правильную
  последовательность включения приёма и разбора статуса:
  - `dwt_rxenable(DWT_START_RX_IMMEDIATE)` — включение приёмника;
  - опрос `SYS_STATUS` на `RXFCG` (кадр принят) / `RX_TO` (таймаут) /
    `RX_ERR` (ошибка);
  - `dwt_readrxdata()` — чтение принятого кадра;
  - `dwt_rxreset()` — сброс приёмника после ошибки;
  - очистка флагов статуса записью в `SYS_STATUS`.

- `ss_init_main.c` — **initiator** (передатчик) той же схемы. Референс для
  TX-тракта: `dwt_writetxdata` / `dwt_writetxfctrl` / `dwt_starttx`, ожидание
  `TXFRS`.

## Как это ложится на наш проект

- Наш RX_START (0x30) реализует `dwt_rxenable` в непрерывном режиме; событие
  приёма (RXFCG) в нашей архитектуре придёт через IRQ DW1000 → EXTI →
  Radio-обработка. Пример показывает polling-вариант; у нас будет
  прерывание/флаг, но набор регистров и флагов статуса — тот же.
- Диагностика (RSSI/SNR) в этих примерах не раскрыта — формулы пересчёта из
  RX_FQUAL/RX_FINFO берём из DW1000 User Manual (§4.7), их предоставит
  архитектор при реализации GET_SIGNAL_METRICS.
