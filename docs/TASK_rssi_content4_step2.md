## Шаг 1 — ядро станции на content=4 (`tools\mks_station.py`, коммит)
- Per-кадровая логика вынесена в **`Station._ingest(sf, now)`** (тестируемо без serial):
  content-фильтр → `parse_frame` → эхо-фильтр `is_own_echo` → на партнёрском кадре
  `last_heard/src` + `_update_metrics` + реассемблер. Свои (эхо) НЕ дают RSSI/связь.
- **Станция не печатает** — эмитит события через `on_event(dict)` (`text`/`call`/`link`/
  `sent`/`status`/`tx_warn`/`error`); состояние партнёра отдаёт `metrics_snapshot(now)`
  (RSSI/FP/SNR + `fresh`). Это общее ядро для консоли и GUI.
- Поток переведён на `content=4` (`open_and_setup` → `set_stream_mode(4)`); `_ingest`
  фильтрует `sf["content"] != self.content`.
- Консоль (`_console_printer`): `status_line` показывает **живые RSSI/FP_POWER/SNR**
  (вместо «RSSI: н/д (content=3)»); при потере связи — «(устар.)». loopback/чат/beacon/
  call сохранены; `start()/stop()`.
- **Юниты:** свой кадр (эхо) не даёт RSSI/связь; чужой даёт RSSI+TEXT; staleness
  fresh→устар.; `send_text` число фрагментов; loopback слушает себя.

## Шаг 2 — GUI-станция (`tools\mks_station_gui.py`, коммит)
tkinter поверх `Station` (подписка `on_event` → tk-очередь → рисование из главного треда):
- **Сигнал-шкала RX_LEVEL** — полоски как индикатор сети смартфона (`dbm_to_bars`,
  пороги `RSSI_FULL=-75` / `RSSI_EDGE=-95` в константах — **подобрать на железе**); цвет
  край→красный / середина→жёлтый / много→зелёный; при потере связи гаснет.
- **FP_POWER / SNR** — числами рядом (dBm / dB), с пометкой «(устар.)» при потере.
- **Индикатор связи** цветом (СВЯЗЬ ЕСТЬ/ПОТЕРЯНА/—) + «N с назад».
- **Свой src_id и адрес партнёра**.
- **Лог сообщений** — принятые TEXT с меткой отправителя и статусом полное/неполное
  (потерянные фрагменты явно); свои отправленные («→ я») тоже видны.
- **Поле ввода с живым счётчиком** — байты UTF-8 и число фрагментов `ceil(len/119)`,
  обновляется по мере набора (педагогика: как сообщение бьётся на кадры).
- **Кнопка «Вызов»** + баннер и звонок терминала при входящем CALL.
- Аргументы: `--port --src-id --mode --loopback`.
- **Юниты:** `dbm_to_bars` (full/edge/монотонность), `frag_count` (граница 119),
  `bars_color`; GUI build smoke (счётчик 250 Б → 3 фрагм., лог/вызов через `_handle`).

## Приёмка захода 2 — на железе (ВАГАН)

### Шаг A — loopback одной платой (RSSI живёт в GUI)
```
python tools\mks_station_gui.py COM3 --src-id 1 --loopback
```
- Эхо-фильтр выкл (слушаем себя) → метрики своих кадров идут в шкалу: **сигнал-шкала и
  RSSI/FP/SNR оживают** (значения loopback M1→M2). Ввод текста → он появляется в логе
  как «→ я» и (в loopback) возвращается как `[1] …`. Счётчик байт/фрагментов реагирует
  на набор (ввести >119 симв. → «2 фрагм.»). «Вызов» → баннер+звонок.
- Консоль (быстрая отладка): `python tools\mks_station.py COM3 --src-id 1 --loopback` —
  `/status` показывает живые RSSI/FP/SNR.

### Шаг B — две платы / две машины (шкала реагирует на отход)
```
машина A:  python tools\mks_station_gui.py COM3 --src-id 1
машина B:  python tools\mks_station_gui.py COM4 --src-id 2
```
- Двусторонний чат **без само-эха** (эхо-фильтр по src). RSSI/шкала показывают **уровень
  ПАРТНЁРА** (только чужие кадры).
- **Замер рабочей зоны:** отходить с ноутбуком B — **шкала падает** (полоски гаснут,
  цвет краснеет), на границе связь теряется (индикатор красный, метрики «устар.»);
  вернуться — шкала растёт. Это и есть количественный измеритель (dBm), а не бинарный.
- Проверить пороги `RSSI_FULL/RSSI_EDGE` на реальных уровнях — при необходимости
  подправить константы (это единственная «настройка на железе»).
- «Вызов» с A → у B баннер+звонок. Длинный текст на границе → неполное сообщение с
  пометкой потерянного фрагмента.

**Отчёт Вагана:** оживает ли RSSI/шкала в loopback; на двух платах — падает ли шкала
при отходе и восстанавливается при возврате; адекватны ли пороги; чат без эха; счётчик
фрагментов; CALL; неполные сообщения.
